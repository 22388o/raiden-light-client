Special custom codec with validates a type equivalent to: ConfirmableAction & ValueOf i.e. a ConfirmableAction intersected with an union of any possible registered RaidenAction. This is needed in order to properly handle members of actions which require special encoding/ decoding logic, like BigNumbers, otherwise when decoding actions (e.g. from RaidenState['pendingTxs']), the members would be kept but not decoded properly.
